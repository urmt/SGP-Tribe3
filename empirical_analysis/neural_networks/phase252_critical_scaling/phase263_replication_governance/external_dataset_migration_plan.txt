2. MIGRATION STEPS
================================================================================

STEP 1: Dataset admission
  - Verify metadata completeness (dataset_admission_rules.txt Section 2)
  - Check sampling requirements (Section 1)
  - Verify no prohibited preprocessing (Section 4)
  - Run admission decision tree
  - Assign dataset ID: ds{NNNNN}
  - Create directory structure
  - Archive README and metadata

STEP 2: Analysis script development
  - Use frozen estimator definitions from parameter_lock_registry.json
  - Scripts must use identical R, chi, Binder, tau definitions
  - Scripts must use numpy default_rng(seed) with documented seed
  - No custom estimators; no alternative order parameters
  - Archive all scripts in analysis/ subdirectory

STEP 3: Figure generation
  - Generate all figures using Phase 252-256 naming convention
  - Figure names: {dataset_id}_{structure}_{N}.png
  - Minimum resolution: 800x600 PNG
  - All figures archived in outputs/ and figures/
  - SHA256 hashes computed for all figures

STEP 4: Cross-dataset stability evaluation
  - Apply cross_dataset_stability_criteria.txt for each structure (F1-F8)
  - Record PASS / INCONCLUSIVE / FAIL for each criterion
  - Compute overall verdict: SR / PR / IR / FR
  - Document verdict in replication_report.txt

STEP 5: Failure analysis (if any FAIL)
  - Complete failure_report.txt for each FAIL criterion
  - Classify failure: genuine physical, methodological, implementation, audit
  - Determine if tier downgrade applies (replication_failure_conditions.txt)
  - Determine if audit suspension applies

STEP 6: Audit chain completion
  - Complete audit_trace.txt: script -> figure -> claim -> verdict
  - Independent second analyst verification
  - Complete verification_checklist.txt

STEP 7: Manuscript integration
  - Write ds000114_summary.txt (one paragraph)
  - Update figure_manifest.csv with ds000114 entries
  - Generate ds000114_claim_alignment.txt
  - Generate ds000114_language.txt (approved language for this dataset)

STEP 8: Phase 263 governance archive
  - Update phase263_master_log.txt with dataset entry
  - Archive migration_completion_report.txt
  - Ensure all files are SHA256-verified

3. EXPECTED OUTPUTS FOR ds000114
================================================================================

Primary outputs (required for any dataset):
  1. R_vs_K curves for all N values (at least N=16,32,64)
  2. Finite-size sharpening figure (R vs K overlaid for all N)
  3. Hysteresis forward/backward sweep figure + loop_area value
  4. Binder U4(K) curves for all N
  5. Susceptibility chi(K) curves for all N
  6. Chi peak scaling figure (chi_peak vs N, log-log fit)
  7. Critical slowing tau(K) figure + peak_ratio

Secondary outputs (if applicable):
  8. Topology comparison (A2A vs alternative)
  9. Frequency comparison (if multiple distributions tested)

Expected format:
  - All figures: PNG, 800x600 minimum, labeled axes
  - All figures: SHA256-hashed
  - Summary table: K_c estimates (if obtainable), gamma (if obtainable),
    loop_area, peak_ratio, overall verdict

Expected verdict range: SR to FR
  - SR: geometric structures fully reproduced
  - PR: transition confirmed, partial structure agreement
  - IR: transition observed, structures inconclusive
  - FR: transition geometry differs from Phase 252-256

================================================================================
4. AUDIT INHERITANCE CHAIN
================================================================================

Each dataset inherits audit structure from Phase 252-262:

Phase 252-256: Single-dataset analysis (seed 42, A2A + ER, Gaussian + Uniform)
  -> phase257_audit_package/ (verdicts for all figures)
  -> phase258_filesystem_verification.txt (SHA256 hashes)
  -> phase259_manuscript_foundation/ (claim governance, tier structure)

Phase 263+: Cross-dataset replication
  -> Each dataset: ds{NNNNN}/audit/audit_trace.txt
  -> phase263_master_log.txt (aggregated verdicts across datasets)
  -> phase263_replication_governance/dataset_admission_rules.txt
  -> phase263_replication_governance/cross_dataset_stability_criteria.txt
  -> phase263_replication_governance/replication_failure_conditions.txt

Audit inheritance rules:
  - Phase 252-256 claims remain valid for their original datasets
  - Replication datasets are assessed independently
  - No replication dataset can "overrule" Phase 252-256 claims
  - Cross-dataset patterns (multiple SR verdicts) strengthen generalization
  - Cross-dataset conflicts (SR vs FR) trigger audit council review

Governance inheritance:
  - parameter_lock_registry.json applies to ALL datasets
  - results_language_constraints.txt (Phase 261) applies to ALL datasets
  - results_limitations_inline.txt (Phase 261) applies to ALL datasets
  - Forbidden shortcuts (Phase 262) apply to ALL datasets

Tier inheritance:
  - Tier 1 claims (C1-C6) are strengthened by SR verdicts from multiple datasets
  - Tier 1 claims are NOT invalidated by a single FR verdict
  - Tier 2 claims (C7-C10) are more fragile — each dataset may produce
    different gamma values; C7 and C10 are dataset-specific

================================================================================
5. MANUSCRIPT UPDATE POLICY
================================================================================

Adding a new dataset to the manuscript requires:

5.1 Dataset summary paragraph:
    One paragraph per dataset summarizing:
    - Dataset source and characteristics
    - Key parameters (N, K range, topology, frequency)
    - Verdict (SR/PR/IR/FR)
    - Agreement or disagreement with Phase 252-256

5.2 Figure integration:
    - New figures from replication dataset are added to manuscript
    - Phase 252-256 figures remain as primary evidence
    - Replication figures are presented as secondary evidence
    - Label all replication figures as "(Replication: ds{NNNNN})"

5.3 Claim strengthening:
    - SR verdict from replication dataset: "This behavior is also observed
      in an independent dataset (ds{NNNNN})."
    - PR verdict: "A partial replication confirms the transition geometry
      in an independent environment."
    - IR verdict: "The transition is confirmed in an independent dataset;
      further characterization is needed."
    - FR verdict: "A replication attempt produced different transition
      geometry, warranting further investigation."

5.4 Claim downgrade:
    - Downgrade only if 2+ datasets produce FAIL on same criterion
    - Downgrade requires audit council review
    - Downgrade document is archived in phase263_replication_governance/

5.5 New claims from replication:
    - Replication may reveal NEW geometric structures not in Phase 252-256
    - NEW claims must follow Phase 261 governance (tier assignment,
      language constraints, limitations)
    - NEW claims are Tier 2 unless independently verified
    - NEW claims cannot contradict existing Tier 1 claims without
      strong evidence and audit council approval

5.6 Cross-dataset generalization:
    - Generalization requires at least 3 datasets with SR or PR verdict
    - Generalization language: "Across multiple independent datasets,
      the transition geometry is consistent with..."
    - Generalization does NOT claim universality
    - Generalization does NOT claim universal critical exponent
    - Generalization is limited to structural features (sharpening,
      Binder dip, hysteresis absence) that survive independent conditions
